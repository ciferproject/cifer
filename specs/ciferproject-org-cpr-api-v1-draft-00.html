1. Introduction

The [CIFER] initiative includes work to define a set of shared APIs for core functions in identity and access management. The Central Person Registry (CPR) is an open source implementation of an Identity Registry. The API defined in this document covers the set of basic operations on person data in the CPR person store.

1.1. Definitions

Base URL:
The CPR REST API invocations defined below are always relative to a Base URL. The Base URL MUST NOT contain a query string as clients may append additional path information and query parameters as part of the request. Example Base URL: https://api.example.com/cpr/v1/

4. API

4.1. People

4.1.1. Add person

POST /people

All input parameters are provided in the body of the POST in a JSON encoding

Parameter Value Description Parameter Type Data Type
updatedBy Required Contains the entity that is updating/creating the record body string

Response content type: application/json

4.1.1.1. Response: Exact match found:

Status code: 200 OK

Example Response Body:

{
    "resource":{
            "personId":"121234",
            "userid":"xyz123",
            "psuId":"912345678",
            "matchType":"USERID_EXACT",
            "uri":"v1/people/121234"
    },
    "resourceMetadata": {
        "lastModified":"2012-10-04T03:10:14.123Z",
        "baseUri":"https://example.edu/cprapi",
        "statusCode":"Exact match found."
    },
    "responseMetadata": {
        "responseTimestamp:"2012-10-04T03:10:14.123Z",
        "responseTime":"2",
        "httpStatusCode":"200",
        "serverVersion":"1.0"
    }
}

4.1.1.2. Response: New person created

Status code: 201 Created

Example Response Body:

{

            "personId":"121234",

        "responseTimestamp:"2012-10-04T03:10:14.123Z",
        "responseTime":"2",
